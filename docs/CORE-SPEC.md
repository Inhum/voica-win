# Voica — Core Spec (кросс-платформенная спека поведения)

Канонический документ поведения Voica, **общий для всех платформ**. Референс-реализация —
macOS (`Inhum/voica`, Swift/AppKit). Порт под Windows (`Inhum/voica-win`, C#/.NET WPF) обязан
воспроизводить это поведение; платформенные отличия (UI-фреймворк, системные API, дефолты
хоткея, пути хранения) явно отмечены.

**Правило паритета:** добавил/изменил фичу в одной ОС → сначала правишь эту спеку, потом
зеркалишь во вторую реализацию. Матрица паритета — в конце файла.

---

## 1. Суть приложения

Фоновое приложение в трее/меню-баре для macOS и Windows. По глобальному хоткею записывает
речь с микрофона, распознаёт её **с пунктуацией** и вставляет текст в активное поле (или
показывает в окне). Движок распознавания — на выбор: **облако** (Groq Whisper, BYO-key) или
**локальный офлайн** (на устройстве, без сети и ключа; платформо-специфично, см. §2.5).

## 2. Groq Speech-to-Text (контракт запроса)

- **Endpoint:** `POST https://api.groq.com/openai/v1/audio/transcriptions`
- **Модель:** `whisper-large-v3-turbo`
- **Auth:** заголовок `Authorization: Bearer <key>`
- **Тело:** `multipart/form-data`, поля:
  - `file` — аудиофайл записи (см. §3)
  - `model` = `whisper-large-v3-turbo`
  - `response_format` = `verbose_json` (даёт `text` + `language` + `duration`)
  - `temperature` = `0`
  - `prompt` = словарь терминов (**только если непустой**, см. §6)
  - `language` — **не задаём** (Whisper определяет сам; важно для смеси рус+англ)
- **Ответ (200):** JSON с полями `text` (обязателен), `language`, `duration`. `text` тримить.
- **Таймаут:** 120 c.
- **Ошибки → сообщения пользователю:** 401 «ключ отклонён, проверь в Настройках»,
  413 «запись слишком длинная, разбей», 429 «лимит Groq, подожди», прочее «Groq вернул N: …»
  (тело обрезать до ~200 симв). Сетевые ошибки — показать текст ошибки.

### Валидация ключа
- `GET https://api.groq.com/openai/v1/models` с `Authorization: Bearer <key>`, таймаут 20 c.
- 200 → ключ рабочий; 401 → отклонён; иначе → `HTTP N`.

## 2.5. Локальный офлайн-движок распознавания (опциональный, платформо-специфично)

Альтернатива облаку: распознавание целиком **на устройстве**, без интернета и без ключа Groq.
Реализация зависит от платформы (нет общего кросс-платформенного рантайма), поэтому это
**намеренное платформенное отличие** 🔀; общими остаются лишь UX-принципы ниже.

- **Переключатель движка** «Облако (Groq) / Локально (офлайн)» в Настройках (раздел General),
  по умолчанию **облако**. Ключ Groq для локального движка не требуется (онбординг его не
  навязывает, если выбран локальный).
- **Модель качается по требованию** при первом включении: индикатор прогресса, проверка
  **SHA-256**, распакованная модель — вне бандла (рядом с историей, `…/models/`). Пока
  качается — работает облако. Удаление модели — в разделе Data (освобождает место; облако
  продолжает работать, докачать можно снова).
- **Обработка — по целому файлу**, как и облако (§3): пишем всю запись, затем распознаём
  разом (длинные записи чанкуем). **Не** потоковое realtime-распознавание, без VAD.
- **Офлайн-fallback:** если выбрано облако, но сети нет, а локальная модель установлена —
  автоматически распознаём локально и **ненавязчиво уведомляем** пользователя.
- **Первая загрузка модели** после запуска занимает время (разовая инициализация под
  железо) — показываем индикатор «готовлю модель…», иначе похоже на зависание. Модель
  выгружается из ОЗУ после простоя.
- **Словарь-подсказка (§6) — только облако** (это фича Whisper `prompt`); **ИИ-исправление
  терминов (§6.1) работает с обоими движками** (нужны ключ и сеть).
- В историю пишется имя движка/модели в колонке `model`.
- **macOS (референс):** модель **Sber GigaAM v3** (`v3_e2e_ctc`, MIT, русский с пунктуацией),
  сконвертированная в **CoreML**, исполняется на **Neural Engine**. Конвейер на Swift без
  Python: мел-фронтенд → CoreML → CTC-декод. Раздаётся **скомпилированной** (`.mlmodelc`,
  ~400 МБ) как release-ассет отдельного тега репозитория. Ограничение: англицизмы модель
  пишет кириллицей (компенсируется ИИ-исправлением §6.1).
- **Windows:** пока не реализовано (CoreML/ANE — технология Apple). Паритет потребует иного
  рантайма (напр. ONNX Runtime / whisper.cpp с GigaAM или Whisper) — будущая отдельная работа.

## 3. Запись аудио

- **Требование:** моно, **16 кГц**, компактно. Цель — быстрый аплоад в Whisper.
- macOS: `.m4a` (AAC, 16 кГц моно, 32 кбит/с) через `AVAudioRecorder`.
- Windows: допустимо **WAV 16 кГц моно** (PCM) через NAudio/WASAPI — Groq принимает WAV.
  (AAC/m4a — опционально позже, если важен размер.) Контейнер платформо-зависим, но
  параметры сигнала (моно/16 кГц) едины.
- **Мин. длительность 0.3 c** — более короткие записи считаем случайным нажатием: файл
  удаляем, ничего не отправляем.
- Временный файл записи удаляется после переноса в историю (или при ошибке/пустом ответе).

## 4. Хоткей и режимы диктовки

- Два режима: **PTT (hold to talk)** — держишь клавишу, говоришь, отпускаешь; **Toggle** —
  нажал начать / нажал закончить. Оба режима — на всех платформах, выбор в Настройках.
- Клавиша-модификатор выбирается из списка (Настройки).
- **Дефолты (платформо-зависимы, намеренно):**
  - macOS: режим **PTT**, клавиша **Right Option** (keyCode 61).
  - Windows: режим **Toggle**, клавиша **Right Alt**.
- Требует системного доступа для глобального перехвата: macOS — Accessibility;
  Windows — низкоуровневый keyboard hook / `RegisterHotKey` (спец. разрешение не требуется).
- Состояние: `idle → recording → transcribing → idle`. Иконка трея отражает состояние
  (запись — акцент/пульсация, транскрибация — индикатор).

### Список клавиш-модификаторов
macOS: Right/Left Option, Right/Left Command, Right Control, Function.
Windows-аналоги: Right/Left Alt, Right/Left Ctrl, Right/Left Win (маппинг по смыслу),
дефолт — Right Alt.

## 4.1. Меню трея / меню-бара (ручной запуск без хоткея)

- Клик по иконке открывает меню. Платформенная механика: macOS — меню открывает **любой**
  клик (левый и правый — стандарт NSStatusItem с меню); Windows — **правый** клик открывает
  контекстное меню (стандарт трея), левый клик может дублировать открытие меню.
- Пункты меню (в этом порядке):
  1. **«Диктовать»** — ручной старт/стоп диктовки без горячей клавиши. Семантика всегда
     toggle, независимо от выбранного режима хоткея: `idle` → старт, `recording` → стоп,
     `transcribing` → игнорировать.
  2. — разделитель —
  3. **«История…»**, **«Настройки…»** (macOS: шорткат ⌘,), **«О программе»**,
     **«Проверить обновления…»** (поведение и трансформация пункта — §10).
  4. — разделитель —
  5. **«Завершить»** (macOS: ⌘Q).

## 5. Вывод результата (Output)

- Два режима, выбор в Настройках:
  - **`insert` (по умолчанию)** — положить текст в буфер обмена **и** синтезировать вставку
    в активное поле (macOS — ⌘V, Windows — Ctrl+V через `SendInput`).
  - **`window`** — показать редактируемое окно результата (как исторически).
- **Текст ВСЕГДА копируется в буфер** — фолбэк, даже в режиме `insert`.

## 6. Словарь терминов (Vocabulary)

- Freeform-строка в Настройках (многострочное поле). Слова/названия/жаргон, которые Whisper
  часто коверкает.
- Готовится в поле `prompt`: **тримминг**; если пусто → `prompt` не отправляем; если длиннее
  **бюджета 800 символов** → берём **хвост** (Whisper учитывает только последние ~224 токена).
- Это **подсказка, не жёсткая замена**: слова-омофоны обычных (напр. `voica`↔«voice») может не
  перебить — это ожидаемо (для железной замены см. ROADMAP: LLM-постобработка).

## 6.1. ИИ-исправление терминов (LLM-постобработка, opt-in)

- Тумблер в Настройках (раздел словаря), **по умолчанию выкл**. Работает только при непустом
  словаре. Применяется к результату **обоих** движков (облако и локальный, §2.5).
- После распознавания текст отправляется в Groq chat-модель:
  `POST https://api.groq.com/openai/v1/chat/completions`, модель **`llama-3.3-70b-versatile`**,
  `temperature: 0`, `max_completion_tokens: 4096`, таймаут **20 с**.
- **Модель может измениться:** Groq периодически удаляет/переименовывает модели (в 0.9.3
  исчез прежний `qwen/qwen3-32b` → 404). Модель захардкожена; verify (ниже) обязан отличать
  **404** (модель удалена/переименована — «обновите приложение») от **403** (заблокирована
  в Groq-org — «разрешите в console.groq.com → Settings → Limits»).
- Промпт (смысловой контракт, дословно см. референс `GroqClient.postProcessPrompt`):
  исправить только искажённые варианты словарных терминов, согласуя с падежом/контекстом;
  слово, совпадающее со словарным термином в другом регистре, — правильное, регистр не менять;
  при неоднозначности выбирать термин по **фонетической близости**; больше ничего не менять.
- **Fail-open:** любая ошибка/не-2xx/таймаут/пустой ответ → доставляется исходный текст Whisper.
  Постобработка никогда не блокирует диктовку.
- Пока идёт постобработка, состояние остаётся `transcribing` (иконка показывает работу).
- В историю пишется **финальный** (исправленный) текст.
- Документировать для пользователя: словарь задаёт каноническое написание (регистр!), и если
  Groq-org пользователя ограничивает модели — chat-модель нужно разрешить в консоли Groq.
- **Verify доступности модели (UX):** при включении тумблера (и при открытии настроек с уже
  включённым) — лёгкий пробный запрос к chat-модели; нативный статус рядом: зелёный
  «модель доступна», при **403** — «разрешите модель в console.groq.com → Settings → Limits»,
  при **404** — «модель недоступна (Groq мог переименовать/убрать) — обновите приложение»,
  прочие ошибки — их текст. Не блокирует UI.

## 7. История (SQLite)

- Таблица `transcriptions`:
  `id INTEGER PK AUTOINCREMENT, created_at INTEGER (unix-секунды), text TEXT NOT NULL,
  language TEXT, duration_sec REAL, audio_filename TEXT, model TEXT`.
- Хранилище **вне бандла/exe**, чтобы переживать обновления:
  - macOS: `~/Library/Application Support/com.ushakov.voica/` (`history.sqlite`, `audio/*.m4a`)
  - Windows: `%APPDATA%\Voica\` (`history.sqlite`, `audio\*.wav`)
- Функции: список (новые сверху), повторное копирование, воспроизведение аудио, удаление записи.
- **Потокобезопасность:** весь доступ к БД сериализован (serial queue на macOS / эквивалентный
  замок или выделенный поток на Windows). Все методы БД идут через него.

## 8. Хранение аудио и retention

- Тумблер «Хранить аудиозаписи» (по умолчанию вкл). Если выкл — аудио не сохраняется, остаётся
  только текст.
- «Удалять аудио старше N дней», по умолчанию **30**, `0` = хранить всегда.
- Чистка при запуске: удаляет аудиофайлы старше N дней и **обнуляет `audio_filename`**; текст
  записи остаётся.

## 9. Хранение ключа Groq

- В **защищённом локальном файле**, не в коде и не в репозитории:
  - macOS: файл `credentials`, права `0600`.
  - Windows: файл, защищённый **DPAPI** (или Windows Credential Manager).
- Fallback для разработки: переменная окружения `GROQ_API_KEY` (если файла нет).
- Сохранение ключа — кнопкой Save; остальные настройки применяются сразу.

## 10. Проверка обновлений

- Анонимный `GET https://api.github.com/repos/<OWNER>/<REPO>/releases/latest`, заголовки
  `Accept: application/vnd.github+json` и `User-Agent: Voica`. **Каждая ОС проверяет СВОЙ репо:**
  macOS → `Inhum/voica`, Windows → `Inhum/voica-win`.
- Сравнение версий: нормализовать тег (убрать ведущее `v`), покомпонентно (`0.10.0 > 0.9.0`).
- При запуске — **не чаще раза в сутки** (кэшировать момент последней проверки); тумблер
  «Проверять при запуске» (по умолчанию вкл). Пункт меню «Проверить обновления…» — по клику.
- Если версия новее — предложить открыть **страницу релиза** (пункт меню меняется на «Скачать
  обновление X.Y.Z», алерт при ручной проверке). **Никогда не качает/не ставит сам.**

## 11. Удаление всех данных и сброс настроек

- Кнопка «Удалить все данные…» с подтверждением **случайной фразой** (`delete-xxxx`): удаляет
  все записи, аудио, ключ и настройки; сбрасывает к дефолтам.
- Кнопка «Сбросить настройки…» (обычное подтверждение OK/Cancel): возвращает настройки к
  дефолтам, **сохраняя** ключ, историю, аудио и словарь терминов (словарь — пользовательский
  контент, не настройка).
- UI словаря: живой счётчик символов `N / бюджет` (бюджет prompt из §6), при превышении —
  предупреждающий цвет (отправляется хвост).

## 12. Локализация, версия, приватность, самотест

- **UI на английском и русском**, по языку системы.
- **Версия** читается из бандла/сборки (единый источник правды), semver.
- **Приватность:** нет бэкенда, нет телеметрии. Сеть только к `api.groq.com` (облачная
  транскрибация и/или ИИ-исправление) и `api.github.com` (опциональная проверка обновлений;
  плюс разовая докачка локальной модели из release-ассета, §2.5). При **локальном** движке
  без ИИ-исправления аудио и текст **никуда не уходят** — обработка целиком на устройстве.
- **Самотест** без сети/GUI (напр. флаг `--test-all`), покрывает чистую логику: сравнение
  версий, подготовку `prompt` из словаря, round-trip хранилища ключа и настроек, insert/delete
  истории, маппинг клавиш, конкурентный доступ к БД. Восстанавливает изменённое состояние.
